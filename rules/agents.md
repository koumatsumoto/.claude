# エージェントのオーケストレーション

## 利用可能なエージェント

配置場所: `~/.claude/agents/`

| エージェント         | 目的             | 利用タイミング               |
| -------------------- | ---------------- | ---------------------------- |
| planner              | 実装計画         | 複雑な機能、リファクタリング |
| architect            | システム設計     | アーキテクチャ判断           |
| code-reviewer        | コードレビュー   | コードを書いた後             |
| security-reviewer    | セキュリティ分析 | コミット前                   |
| build-error-resolver | ビルドエラー修正 | ビルド失敗時                 |
| refactor-cleaner     | デッドコード清掃 | コードメンテナンス           |
| doc-updater          | ドキュメント     | ドキュメント更新             |

## 即時エージェント利用

ユーザーの明示的な指示は不要:

1. 複雑な機能リクエスト - **planner** エージェントを使う
2. コードを書いた直後 - **code-reviewer** エージェントを使う
3. アーキテクチャ判断 - **architect** エージェントを使う

## 並列タスク実行

独立した作業は **必ず** 並列で実行する:

```markdown
# GOOD: 並列実行

3 エージェントを並列起動:

1. エージェント 1: auth.ts のセキュリティ分析
2. エージェント 2: cache システムの性能レビュー
3. エージェント 3: utils.ts の型チェック

# BAD: 不要な逐次実行

まずエージェント 1、それからエージェント 2、最後にエージェント 3
```

## 多視点分析

複雑な問題では、役割分割のサブエージェントを使う:

- 事実確認のレビュアー
- シニアエンジニア
- セキュリティ専門家
- 一貫性レビュアー
- 冗長性チェッカー
