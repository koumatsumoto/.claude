# セキュリティガイドライン

## 事前チェック

コミット前に次を確認する。

- [ ] シークレットのハードコードがない
- [ ] 入力検証と出力エスケープがある
- [ ] 認証/認可が適切に適用される
- [ ] 危険なクエリ/コマンド実行がない
- [ ] エラーやログに機密情報を出していない

## 実装ルール

- 秘密情報は環境変数またはシークレットストアで管理する
- 入力は型と制約を検証してから利用する
- 権限チェックは API/操作単位で明示する
- 依存関係の脆弱性は定期的に監査する

## ブロッカー

以下を検出した場合、修正完了まで出荷/コミットを止める。

- ハードコードされたシークレット
- 認可欠如による不正アクセス
- 非検証入力の直接利用
- 機密情報の露出ログ

## インシデント対応

1. 影響範囲を特定して一時封じ込め
2. 根本原因を修正
3. 露出した秘密情報をローテーション
4. 再発防止の検査を追加
