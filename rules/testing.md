# テスト要件

## 最低テストカバレッジ: 80%

テスト種別（すべて必須）:

1. **ユニットテスト** - 個々の関数、ユーティリティ、コンポーネント
2. **統合テスト** - API エンドポイント、データベース操作
3. **E2E テスト** - 重要なユーザーフロー（Playwright）

## テスト駆動開発（TDD）

必須ワークフロー:

1. テストを先に書く（RED）
2. テストを実行 - 失敗するはず
3. 最小の実装を書く（GREEN）
4. テストを実行 - パスするはず
5. リファクタリング（IMPROVE）
6. カバレッジを検証（80%+）

## テスト失敗のトラブルシューティング

1. **tdd-guide** エージェントを使う
2. テストの分離を確認
3. モックが正しいか確認
4. テストではなく実装を直す（テストが誤りの場合を除く）

## エージェント支援

- **tdd-guide** - 新規機能にプロアクティブに使用し、テストファーストを徹底
- **e2e-runner** - Playwright E2E テストの専門エージェント
