# コーディングスタイル

## 基本方針

- 読みやすさと保守性を優先する
- 破壊的変更より小さな差分を優先する
- 既存コードの規約に合わせる

## 設計ルール

- オブジェクト/配列はイミュータブルに扱う
- 関数は責務を 1 つに保つ
- 深いネストを避け、早期 return を使う
- マジックナンバーは定数化する

## ファイル運用

- 目安: 1 ファイル 300 行以内（最大 800 行）
- 大きいファイルは機能単位で分割する
- 命名はドメイン語彙を優先する

## 品質ルール

- 例外は握り潰さず、文脈を付けて扱う
- ユーザー入力は必ず検証する
- `console.log` の残置を避ける
- 不要なコメントより、意図が伝わるコードを優先する

## 完了チェック

- [ ] 仕様どおりに動作する
- [ ] 主要ケースのテストがある
- [ ] 可読性を下げる重複がない
- [ ] セキュリティ上の問題がない
