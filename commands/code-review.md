# コードレビュー

未コミット変更に対する包括的なセキュリティ/品質レビュー:

1. 変更ファイルを取得: git diff --name-only HEAD

2. 各ファイルで以下を確認:

**セキュリティ問題（CRITICAL）:**

- ハードコードされた資格情報、API キー、トークン
- SQL インジェクションの脆弱性
- XSS 脆弱性
- 入力バリデーション欠如
- 危険な依存関係
- パストラバーサルの危険

**コード品質（HIGH）:**

- 50 行超の関数
- 800 行超のファイル
- 4 階層超のネスト
- エラーハンドリング不足
- console.log の存在
- TODO/FIXME コメント
- 公開 API の JSDoc 不足

**ベストプラクティス（MEDIUM）:**

- ミューテーションパターン（immutable を使う）
- コード/コメントでの絵文字
- 新規コードのテスト不足
- アクセシビリティ問題（a11y）

1. レポートを生成:
   - 重大度（CRITICAL/HIGH/MEDIUM/LOW）
   - ファイル位置と行番号
   - 問題の説明
   - 修正案

2. CRITICAL / HIGH があればコミットをブロック

セキュリティ脆弱性があるコードは絶対に承認しない！
