# Coding Standards Reference

## 1. 命名

- 変数は目的が伝わる名詞を使う
- 関数は動詞 + 対象の形にする
- 真偽値は `is` / `has` / `can` で始める

## 2. イミュータビリティ

- オブジェクト/配列の直接変更を避ける
- 更新はスプレッドや純関数で表現する
- 例外が必要なら理由をコメントで明示する

## 3. エラーハンドリング

- 外部境界では `try/catch` または結果型で扱う
- エラーメッセージに機密情報を含めない
- ログはデバッグに十分な文脈を残す

## 4. 入力検証

- `unknown` を受けてスキーマで検証する
- ホワイトリスト方式を優先する
- バリデーションエラーは利用者向けに整形する

## 5. React

- props の型を明示する
- 状態更新は関数更新を優先する
- 複雑なロジックはカスタムフックへ分離する

## 6. API

- レスポンス形式を統一する
- HTTP ステータスを意味に沿って返す
- 破壊的変更は明示して移行手順を用意する

## 7. テスト

- 仕様（振る舞い）をテストし、実装詳細へ過剰依存しない
- 正常系に加え、異常系・境界値を含める
- 変更箇所に対応するテストを最低 1 つ追加する

## 8. レビュー時の観点

- 可読性
- 安全性
- 保守性
- 既存規約との一貫性
