# Plan

コード変更前に実装計画を作成し、ユーザー承認を得るためのスキル。

## Use When

- 新機能や複雑なリファクタを行う
- 影響範囲が広い
- 要件に曖昧さがある

## Workflow

1. 要件を言い換えてスコープを明確化する
2. リスクと依存関係を洗い出す
3. 実装をフェーズ化する
4. 検証手順（テスト/確認観点）を定義する
5. ユーザー承認を得るまで実装しない

## Output

- 要件整理
- 実装フェーズ
- リスクと対策
- 事前確認事項
